# Repository Workflow

This document defines how the repository is maintained throughout the learning journey.

---

## After Every Lesson

Update:

- PROGRESS.md
- JOURNAL.md
- Relevant module notes
- Resources (if new references were used)
- Commands (if new commands were learned)
- FAQ (if important questions arose)
- GLOSSARY (if new terminology was introduced)

---

## After Every Module

Create or update:

- Module Summary
- Cheatsheet
- Module Diagram(s)
- Project Updates
- Module README

Review all lessons before marking the module complete.

---

## Notes Guidelines

Notes should explain concepts in my own words.

Avoid copying documentation.

Focus on:

- Why
- How
- Trade-offs
- Production Usage

---

## Resources

Only add resources that are genuinely useful.

Prefer:

- Official documentation
- Books
- Technical blogs
- Research papers
- High-quality videos

---

## Commands

Record only commands that have been used and understood.

Each command should include:

- Purpose
- Syntax
- Example
- Notes

---

## Diagrams

Whenever a concept involves communication, architecture, or request flow, create a diagram.

Examples:

- Request lifecycle
- Packet flow
- Architecture
- Deployment
- Networking

---

## Journal

At the end of every lesson answer:

1. What did I learn?
2. What surprised me?
3. What remains unclear?
4. Can I explain this without notes?

---

## Progress

A lesson is complete only if:

- Theory is understood
- Hands-on exercise is completed
- Notes are written
- Reflection is completed

Only then should it be marked as completed in ROADMAP.md and PROGRESS.md.
