# Systems From Scratch

> A hands-on journey to understand modern software systems from first principles.

---

## Vision

This repository documents my journey to deeply understand how modern software systems work—from the moment a user enters a URL in a browser to the point where a production-scale application serves millions of users.

The goal is not to memorize tools or commands. The goal is to understand the reasoning behind every layer of the software stack and how those layers work together.

---

## Objectives

By the end of this journey, I aim to understand:

- Computer Networking
- HTTP & HTTPS
- Linux
- Backend APIs
- Reverse Proxies
- Docker
- Kubernetes
- Service Communication
- Databases
- Messaging Systems
- Observability
- Production Architecture

More importantly, I want to understand how these components interact to build reliable, scalable, and maintainable systems.

---

## Learning Philosophy

Every concept is learned by answering four questions:

1. Why does this exist?
2. What problem does it solve?
3. How does it work internally?
4. How is it used in production?

Learning does not stop at theory. Every major concept is reinforced with:

- Notes
- Hands-on labs
- Experiments
- Diagrams
- An evolving project

---

## Repository Structure

- `modules/` - Learning material organized by module.
- `projects/` - The evolving project built throughout the course.
- `diagrams/` - Architecture and flow diagrams.
- `commands/` - Useful command references.
- `resources/` - Books, articles, videos, and documentation.
- `cheatsheets/` - Quick revision material.
- `experiments/` - Small prototypes to validate concepts.

## The Project

Throughout this journey, a single Book Store application will evolve from a simple local application into a production-ready distributed system.

Each module introduces architectural improvements that solve real engineering problems.
